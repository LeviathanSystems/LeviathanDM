# Contributing

We welcome contributions to LeviathanDM!

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## Code Style

- Follow existing code style
- Use meaningful variable names
- Add comments for complex logic
- Update documentation as needed

## Building

See the [README](https://github.com/LeviathanSystems/LeviathanDM#building) for build instructions.

## Testing

Test your changes in both nested (cage) and TTY environments.
